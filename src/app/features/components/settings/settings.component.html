Notes/Observations:
  • I noticed that APi response from server is having inconsistent naming conventions i.e all properties are in pascal case but property accordion is of camel case.
  • Since Js/Angular follows camel casing, I have created a utility service and added function in it which converts the response to came case and adheres to UI coding standards and the model created in angular can have came case properties
  • There are typos in property names from server response i.e CategoyHeading , CategoyText , so word category is misspelled in API response.
  • Added code to configure server url dynamically, which makes it deployable using any CI/CD tools.
  • Added spinner and Toastr while loading and saving for better user experience
